# Seminar 2

* Создание веток

1. Для создания новой ветки используется команда *git branch_name*, где *branch_name*-название новой ветки.

2. Для перехода используется команда *git checkout branch_name*где *branch_name*-название новой ветки.

3. Обычно если ветка больше не нужна, ее сливают с основной и удаляют.
4. Когда вы работаете с большим количеством веток, можно легко забыть имя нужной, а без имени ветки переключится на нее не получится. Для таких ситуаций существует команда просмотра списка веток - это уже знакомая нам *git branch*, но с другими ключами.
5. По умолчанию *git branch* выводит список локальных веток. С ключами -r, -a можно вывести, соответственно, либо только удаленные ветки, либо все ветки.

* Слияние веток

1. Для осуществления слияния веток используется команда *git marge branch_name*





* Конфликты при слиянии

1. Конфликты при слиянии возникают при возникновении противоречищих коммитов слияния веток.
